Vous connaissez peut-être le principe d'un Twitch Tracker, mais au cas-où, voici un rappel. L'objectif, c'est d'utiliser des données de streamer, qui sont en live sur la plateforme. Ces données, ça peut être le nombre de viewers moyen, le nombre maximal de viewers ƒ?Ý Ces données sont donc stockées dans une base de données, et l'objectif est de les récupérer, et les mettre en page.

## Caractéristiques techniques du projet

### Langages / technologies utilisés

Ce Twitch Tracker se base sur une base de données en MySQL, et les données sont traitées et récupérables via des points d'accès (end-points). Ce traitement se fait en PHP, et pour les mettre en forme, on a utilisé du JavaScript. Evidemment, il y a une base de HTML et CSS pour une mise en page simpliste.

### Combien de temps, qui et quoi ?

Le projet a duré une dizaine à vingtaine d'heures par personne, avec un temps important de formation. Il a été réalisé en groupe de 3, par [Lucas Chalmandrier](https://linkedin.com/in/lucas-chalmandrier), [Florian Mure](https://www.linkedin.com/in/florian-mure-9b335422b/) et moi-même, dans le cadre de notre formation du BUT Informatique Graphique.

## Détails du projet

Le projet est un peu basique, mais il faut bien commencer quelque part. Par ailleurs, nous avons utilisé GitHub pour ce projet, sur lequel vous pourrez trouver plus de détails et les sources du projet.
